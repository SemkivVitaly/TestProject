# Веб-интерфейс (локальный) и доступный функционал

**Интерфейс полностью на русском языке** (главная страница, Bridge UI, параметры SERVO, сообщения и подсказки).

## Локальная работа веб-интерфейса

Веб-интерфейс **работает только в локальной сети** — без выхода в интернет.

1. **Подключитесь к Wi‑Fi точки доступа ESP32**  
   Имя сети (SSID) и пароль заданы в `src/config.h` при первом запуске; далее их можно менять в Bridge UI (`/bridge`) — настройки сохраняются в NVS.

2. **Откройте в браузере**  
   - В режиме **AP**: `http://192.168.2.1`  
   - В режиме **STA** (плата подключена к роутеру): `http://<IP платы>` или `http://esps3.local` (если поддерживается mDNS).

3. **Интернет не требуется** — все страницы и API отдаются самим ESP32. Доступ к веб-интерфейсу возможен только с устройств, подключённых к той же Wi‑Fi сети (точка доступа платы или роутер, к которому подключена плата в режиме STA).

## Доступные страницы и API

| Путь | Описание |
|------|----------|
| `/` | Главная: метрики канала (пакеты rx/tx/обработано, задержка, потери, байты сети), ссылки на LED, серво, параметры, Bridge UI. |
| `/params` | Страница параметров SERVO: запрос с автопилота, отображение и установка SERVO1_REVERS, SERVO3_TRIM, SERVO4_TRIM. |
| `/bridge` | Полный Bridge UI (стиль DroneBridge): настройки Wi‑Fi/режим, сохранение в NVS, перезагрузка. Требует загрузки FS: `pio run -t uploadfs`. Старый адрес `/smd` перенаправляется на `/bridge`. |
| `/status` | JSON: heap, uptime, mavlink_connected, packets_rx/tx. |
| `/api/status` | Расширенный JSON: подключение, пакеты, задержка, потери, байты сети, параметры SERVO, лог. |
| `/api/link` | JSON только метрик канала: packets_sent/received/processed, latency_ms, packet_loss, bytes_network_*. |
| `/api/log` | JSON-массив последних записей лога MAVLink. |
| `/api/params` | GET — параметры SERVO; POST — установка параметров (тело формы). |
| `/api/param_request` | GET — запрос с автопилота всех трёх параметров SERVO. |
| `/api/system/info`, `/api/system/stats`, `/api/settings` | Для Bridge UI: версия, статистика, GET/POST настроек (сохранение в NVS и перезагрузка). |
| `/led/on`, `/led/off` | Включение/выключение LED (пин в config.h). |
| `/servo?angle=0..180` | Установка угла сервопривода. |

## Полный функционал: что реализовано

### Полностью доступно

- **Мост Wi‑Fi ↔ UART**  
  Данные между Mission Planner (или другим GCS) и автопилотом проходят без искажений по TCP (порт 8880) и UDP (порт 14550).

- **Mission Planner и MAVLink**  
  Подключение по TCP или UDP, телеметрия, параметры, команды. Данные корректно считываются и передаются.

- **Парсинг MAVLink**  
  HEARTBEAT (статус подключения), PARAM_VALUE (чтение параметров SERVO), отправка PARAM_REQUEST_READ и PARAM_SET.

- **Параметры SERVO**  
  Чтение и запись SERVO1_REVERS, SERVO3_TRIM, SERVO4_TRIM через веб-страницу `/params` и API.

- **Метрики канала в веб-интерфейсе**  
  Счётчики пакетов (принято/отправлено/обработано), задержка (время с последнего HEARTBEAT), потери (ошибки CRC), байты по сети.

- **Локальный веб-интерфейс**  
  Главная страница, параметры, лог, LED, серво — всё доступно по локальному IP после подключения к Wi‑Fi платы.

- **Bridge UI на `/bridge`**  
  После `pio run -t uploadfs` доступна полная страница Bridge с статистикой и формой настроек. Режим Wi‑Fi (AP/STA), SSID, пароль, hostname и др. сохраняются в NVS и применяются после перезагрузки.

- **Сохранение настроек в Bridge UI**  
  Кнопка «Save Settings & Reboot» сохраняет настройки в NVS и перезагружает плату. При следующей загрузке применяются сохранённые Wi‑Fi режим, SSID и пароль.

- **Настройки Wi‑Fi и режим из веба**  
  В Bridge UI можно задать режим (AP или STA), SSID, пароль, hostname и другие поля формы. После сохранения и перезагрузки плата поднимает Wi‑Fi с новыми параметрами.

### Частично или ограниченно

- **Добавление UDP-клиентов через Bridge UI**  
  Эндпоинты для добавления/очистки UDP-клиентов вызываются, но список не сохраняется между перезагрузками; UDP-клиент определяется по первому пришедшему пакету (например, от Mission Planner).

### Опционально (в config.h)

- **Bluetooth** — раскомментировать `BLUETOOTH` для моста по BT.
- **OTA** — раскомментировать `OTA_HANDLER` для прошивки по Wi‑Fi.
- **Battery saver** — раскомментировать `BATTERY_SAVER` для снижения мощности Wi‑Fi.

## Итог

- **Веб-интерфейс работает только локально**: подключились по Wi‑Fi к плате → открыли в браузере `http://192.168.2.1` (или IP платы в режиме STA).
- **Полностью доступны**: мост, Mission Planner, MAVLink, параметры SERVO, метрики (пакеты, задержка, потери), главная страница, `/params`, LED, серво.
- **Сохранение настроек и Wi‑Fi/режим из веба реализованы**: конфигурация хранится в NVS, при первом запуске используются значения по умолчанию из `config.h`.
